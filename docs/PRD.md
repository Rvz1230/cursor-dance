## 1. 文档目的

## 2. 产品概述

### 2.1 产品定义

CursorDance 是一款面向 PC 浏览器用户的鼠标特效浏览器插件。产品允许用户将鼠标点击、悬停、滚轮、连击、不同光标状态等交互行为进行可视化和个性化配置，实现音效、飘字、粒子、图片、动画、页面反馈、光标替换等效果，并支持方案保存、预设套用、实时预览及后续分享扩展。

### 2.2 产品愿景

让用户以低门槛、高自由度的方式定制浏览器中的鼠标交互表现，让“鼠标行为”从工具属性升级为表达个性、增强趣味与提升演示效果的交互载体。

### 2.3 产品价值

对用户侧：

* 提供鼠标行为可视化与个性化表达能力。
* 降低制作互动特效的门槛，不需要编程即可完成配置。
* 满足娱乐化、直播展示、录屏演示、桌面氛围增强等场景需求。

对产品侧：

* 以插件形态快速验证用户对鼠标特效配置的真实需求。
* 为后续预设市场、方案分享、创作者生态、会员能力提供基础设施。

## 3. 目标用户与使用场景

### 3.1 目标用户

核心用户：

* 喜欢桌面美化、个性化配置的普通浏览器用户。
* 有录屏、直播、演示诉求，希望强化鼠标表现的用户。
* 喜欢尝试插件、动效、视觉反馈类工具的年轻用户。

潜在用户：

* 内容创作者、UP 主、直播用户。
* 教学、演讲、培训等场景中的展示型用户。
* 需要强调点击反馈与交互路径的产品演示人员。

### 3.2 核心使用场景

* 用户希望点击页面时出现粒子、飘字、动画反馈。
* 用户希望悬停时出现轻量视觉提示，增强互动感。
* 用户希望在录屏/直播中强化鼠标动作存在感。
* 用户希望通过预设方案快速切换不同风格。
* 用户希望在不写代码的前提下完成个性化配置。

## 4. 产品目标

### 4.1 MVP 目标

在浏览器插件场景下，完成“鼠标行为触发特效”的最小可用闭环，确保用户可以：

* 安装插件。
* 开启/关闭特效。
* 针对核心鼠标事件配置至少一种反馈效果。
* 实时预览效果。
* 保存并切换本地方案。
* 使用官方预设快速上手。

### 4.2 业务目标

* 验证鼠标特效插件的真实用户兴趣与留存潜力。
* 验证“预设 + 自定义配置”是否为有效使用路径。
* 验证用户最常用的行为触发与最偏好的效果类型。

### 4.3 用户成功标准

用户在首次安装后 3 分钟内可以完成一次特效启用与方案保存；在不查阅说明的情况下理解核心配置逻辑，并能成功看到效果生效。

## 5. MVP 范围

### 5.1 本期必须交付（P0）

#### A. 插件基础能力

* 插件安装与初始化。
* 插件开关（全局启用 / 禁用）。
* 当前页面特效注入与运行。
* 基础权限与运行状态提示。

#### B. 事件触发体系

支持以下基础触发事件：

* 单击。
* 双击。
* 长按（可简化为按下态触发）。
* 悬停。
* 滚轮滚动。

#### C. 效果体系（首批）

支持以下可配置效果类型：

* 飘字。
* 粒子。
* 图片。
* 简单动画。
* 点击高亮/波纹类页面反馈。
* 基础音效。

#### D. 配置能力

支持针对事件配置以下参数：

* 是否启用。
* 触发的效果类型。
* 效果尺寸。
* 持续时间。
* 出现位置策略（跟随鼠标 / 偏移）。
* 数量/密度。
* 文案内容（用于飘字）。
* 素材选择（用于图片/音效）。

#### E. 实时预览

* 在设置页或预览区即时看到配置结果。
* 修改参数后即时生效。
* 提供恢复默认入口。

#### F. 方案管理

* 新建方案。
* 保存方案。
* 复制方案。
* 删除方案。
* 切换方案。
* 内置官方预设方案。

#### G. 设置页信息架构

* 首页/总览。
* 效果配置。
* 方案管理。
* 预设选择。
* 基础设置。

### 5.2 可延后能力（P1）

* 更多触发事件（拖拽、选中文本、右键、组合键等）。
* 更复杂动画编排。
* 用户素材上传与素材库管理增强。
* 方案导入/导出。
* 跨设备同步。
* 更多预设分类。
* 基础数据统计。

### 5.3 后续探索能力（P2）

* 方案分享社区。
* 创作者模板市场。
* 会员专属高级效果。
* AI 辅助生成特效方案。
* 多页面/多站点规则化配置。

## 6. 功能结构

### 6.1 一级模块

1. 插件状态控制
2. 事件与效果配置
3. 实时预览
4. 方案管理
5. 官方预设
6. 基础设置

### 6.2 配置逻辑模型

系统配置关系建议采用：

`方案 -> 事件 -> 效果 -> 参数`

说明：

* 一个方案包含多个事件配置。
* 一个事件可绑定一个或多个效果（MVP 可先限制为一个主效果）。
* 每个效果包含独立参数。

配置示例：

* 方案：Cute Pink
* 事件：单击
* 效果：飘字
* 参数：文案=“啵啵”、字号=16、持续时间=800ms、偏移=8px

### 6.3 当前原型信息架构

为了让 demo 更接近真实产品，当前原型建议采用以下结构：

* Popup 首屏：当前启用中方案预览、总开关、快速方案切换、完整编辑入口。
* 完整设置页：总览 / 方案 / 行为映射 / 鼠标状态 / 素材中心 / 网站规则 / 诊断。
* 行为映射页：明确展示单击、双击、滚轮、悬停、长按分别绑定到左键、右键、滚轮方向或长按触发。
* 鼠标状态页：为 default、pointer、text、help、wait、notAllowed 提供独立图标配置位。
* 素材中心：提供自定义图片、点击音效、飘字文案、粒子密度、音量等可操作模块。

## 7. 核心交互流程

### 7.1 首次使用流程

1. 用户安装插件。
2. 打开插件弹窗或设置页。
3. 默认看到“开启特效”引导。
4. 选择一个官方预设。
5. 在预览区看到效果。
6. 保存为当前方案并启用。
7. 返回网页体验实际效果。

### 7.2 自定义配置流程

1. 用户进入“行为映射”或“素材中心”。
2. 选择一个触发事件，并明确它对应的按键或滚轮方向。
3. 勾选该事件需要的效果模块，例如飘字、音效、粒子、图片、页面反馈、光标替换或长按蓄能。
4. 在“鼠标状态”页配置各状态对应的自定义图标。
5. 在预览区实时查看修改结果。
6. 保存覆盖当前方案或另存新方案。

### 7.3 日常使用流程

1. 用户打开插件弹窗。
2. 快速开关当前方案。
3. 按需切换不同风格方案。
4. 在浏览网页时直接感知效果。

### 7.4 首屏优先原则

* 用户打开插件后，第一眼应看到当前启用中的方案预览。
* 首屏必须同时回答“现在开没开”“现在用哪个方案”“当前方案大概长什么样”。
* 深度配置入口放在完整设置页，避免在弹窗首屏堆叠过多选项。

## 8. 页面结构建议

### 8.1 插件弹窗

目标：满足快速启停与快速切换。

建议包含：

* 总开关。
* 当前方案名称。
* 快速切换预设/方案。
* 进入完整设置页入口。

### 8.2 设置主页

目标：承载完整配置与预览。

建议包含：

* 左侧导航：总览 / 效果配置 / 方案管理 / 预设 / 设置。
* 中间配置区：按事件与效果展开。
* 右侧预览区：实时显示效果。

### 8.3 方案管理页

建议包含：

* 方案列表。
* 新建/复制/删除。
* 启用状态标记。
* 预设与自定义区分。

### 8.4 当前 demo 的推荐结构

* 左侧信息栏：当前状态、方案切换、站点策略、快捷设置、编辑对象。
* 主内容顶部：当前启用中方案预览。
* 主内容中部：Tab 分区。
* Tab 分区建议为：总览 / 方案 / 行为映射 / 鼠标状态 / 素材中心 / 网站规则 / 诊断。

## 9. 数据与配置存储

### 9.1 MVP 存储策略

MVP 阶段采用本地存储：

* 插件设置。
* 当前启用状态。
* 用户自定义方案。
* 最近使用方案。

### 9.2 建议数据结构

```json
{
  "enabled": true,
  "activeSchemeId": "scheme_001",
  "schemes": [
    {

            "offsetX": 0,
            "offsetY": -8

        }
      }
    }
```

## 10. 非目标

以下内容明确不纳入当前主文档对应的 MVP 交付范围：

### 10.1 不做复杂编辑器

* 不提供时间轴级动画编辑器。
* 不支持节点式特效编排。
* 不支持专业级粒子系统编辑。

### 10.2 不做完整社区体系

* 不做用户公开广场。
* 不做点赞、评论、关注等社区功能。
* 不做创作者分成体系。

### 10.3 不做账号体系强依赖

* MVP 不强依赖注册/登录。
* 不做云端同步作为首发前置条件。
* 不做跨设备一致性保障。

### 10.4 不做全平台覆盖

* 当前仅面向 PC 浏览器插件。
* 不覆盖移动端浏览器。
* 不覆盖桌面系统级全局鼠标特效。

### 10.5 不做重度性能监控后台

* MVP 不建设完整数据平台。
* 不做复杂行为分析后台。
* 不做运维级可观测性产品化界面。

## 11. 约束与依赖

### 11.1 技术约束

* 浏览器插件运行环境受宿主页面限制。
* 不同网站的 DOM 结构、样式隔离策略、事件拦截机制可能影响效果表现。
* 音效播放可能受浏览器自动播放策略限制。
* 特效渲染需控制性能开销，避免影响网页正常使用。

### 11.2 产品约束

* 首版需要优先保证“简单好用”，而不是参数无限丰富。
* 特效需要足够明显，但不能过度打扰正常浏览。
* 配置模型必须可理解，避免事件与效果关系过于复杂。

### 11.3 外部依赖

* 浏览器插件商店上架规范。
* 不同浏览器内核的兼容差异。
* 素材版权与音效资源来源管理。

## 12. 风险

### 12.1 性能风险

**描述**：粒子、动画、图片等效果叠加后可能造成页面卡顿、掉帧或输入延迟。
**影响**：影响网页正常浏览体验，降低留存。
**应对**：

* 设置效果数量、频率、持续时间上限。
* 默认预设采用轻量参数。
* 低性能页面自动降级或限制部分效果。

### 12.2 兼容性风险

**描述**：不同网站的样式、Canvas、iframe、Shadow DOM、事件代理机制可能影响注入效果。
**影响**：部分页面效果不生效或表现异常。
**应对**：

* MVP 先保证常见普通网页可用。
* 建立兼容性黑名单/降级策略。
* 将页面异常隔离，避免拖垮整个插件。

### 12.3 交互打扰风险

**描述**：特效过多、音效过强、频繁触发会打扰正常操作。
**影响**：用户快速关闭或卸载插件。
**应对**：

* 默认关闭高干扰效果。
* 提供全局一键关闭。
* 预设分层：轻量/明显/夸张。

### 12.4 配置复杂度风险

**描述**：若参数过多，用户会理解困难、配置放弃率高。
**影响**：从“好玩”变成“难用”。
**应对**：

* MVP 控制参数数量。
* 优先预设驱动，其次再支持细调。
* 通过实时预览降低理解成本。

### 12.5 素材与版权风险

**描述**：图片、音效等素材若来源不清晰，可能引发版权问题。
**影响**：商店审核或商业化阶段受阻。
**应对**：

* 首发仅使用自制、可商用或已授权素材。
* 后续用户上传能力增加审核与免责声明机制。

### 12.6 平台审核风险

**描述**：浏览器商店对插件权限、注入逻辑、资源加载方式可能有限制。
**影响**：上架进度延迟或被拒。
**应对**：

* 首版最小化权限申请。
* 提前按商店规范审查实现方案。
* 避免不必要的数据采集与敏感行为。

## 13. 验收标准

### 13.1 产品级验收标准

1. 用户可以成功安装并启用插件。
2. 用户可以在任意普通网页中看到至少一种鼠标触发特效生效。
3. 用户可以完成“选择预设 -> 调整参数 -> 实时预览 -> 保存方案 -> 启用方案”的完整闭环。
4. 用户可以随时一键关闭所有特效。
5. 用户保存后的方案在插件重启后仍能正确恢复。

### 13.2 功能级验收标准

#### A. 开关能力

* 提供全局启用/禁用开关。
* 关闭后所有特效立即停止触发。
* 重新开启后恢复当前方案。

#### B. 事件触发

* 单击事件可稳定触发。
* 双击事件可稳定触发。
* 悬停事件可稳定触发。
* 滚轮事件可稳定触发。
* 至少 90% 的常见普通网页场景下行为可触发。

#### C. 效果配置

* 用户可为事件选择一种效果类型。
* 修改核心参数后预览结果同步变化。
* 飘字文案、粒子数量/大小、图片素材、音效开关等核心参数可保存。

#### D. 方案管理

* 支持新建、保存、切换、复制、删除方案。
* 官方预设可直接启用。
* 删除方案不会误删内置默认预设。

#### E. 数据持久化

* 当前启用状态持久化。
* 当前方案与自定义方案持久化。
* 浏览器重启后数据不丢失。

### 13.3 性能验收标准

* 在常见内容型网页中，开启默认预设后不出现明显卡顿。
* 高频触发场景下页面仍可正常滚动与点击。
* 单次效果触发不会造成页面长时间冻结。
* 插件异常不会导致宿主页面主流程不可用。

### 13.4 体验验收标准

* 首次使用用户能在 3 分钟内完成启用。
* 配置路径清晰，核心操作不超过 3 层页面进入。
* 预览反馈及时，参数修改后用户可立即感知变化。
* 用户能明确理解当前是否启用、当前使用哪个方案。

## 16. 附录：一句话版本

CursorDance 是一个面向 PC 浏览器的鼠标特效插件，帮助用户通过低门槛配置，让点击、悬停、滚轮等鼠标行为具备可视化、个性化与可演示的反馈效果，并通过预设、方案管理与实时预览实现可持续使用。
