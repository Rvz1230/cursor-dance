# 1. 文档目的

本文档用于解决 Cursor-Dance-Ui-Demo-V3 当前存在的“结构趋于合理，但核心可配置能力仍未真正落地”的问题。

这份文档聚焦的不是：

* 页面是否好看
* 卡片是否过多
* 命名是否统一

而是更关键的下一层：

**用户最在意、最能体现产品价值的配置能力，是否真的能在页面中完成。**

本文档要回答 6 个问题：

1. V3 当前有哪些关键能力缺口。
2. 这些缺口为什么是高优先级问题。
3. 这些缺口应该落在哪个页面或模块。
4. 每个能力模块应包含哪些字段与动作。
5. 哪些属于 MVP 必须补齐，哪些可放到 P1。
6. 前端下一轮 Demo 应优先补哪些真实交互。

---

# 2. 总体结论

V3 相比 V2 已经明显改善了以下问题：

* 中文化统一。
* 顶部导航真实切页。
* Popup 收敛。
* 总览页更轻。
* 诊断页更聚焦异常。

这些改动使 V3 从“结构展示型 Demo”进一步接近“真实产品界面”。

但是，V3 目前暴露出一个更核心的问题：

## 2.1 当前问题已从“结构问题”转向“能力缺失问题”

也就是说：

* 页面开始更像产品了。
* 但用户想真正配置的时候，核心能力还没有接上。

V3 当前最大的问题不是：

* 页面不清楚
* 模块太乱

而是：

**“你让我看到了配置入口，但我最想改的东西还改不了。”**

这会直接损害产品可信度。

---

# 3. 本轮识别出的四类核心能力缺口

经过对 V3 页面结构与现有 PRD、信息架构、线框说明的对照，本轮识别出以下四类高优先级能力缺口：

1. Popup 缺少“当前启用方案预览”。
2. 行为配置页缺少真实可操作的“素材与音效配置模块”。
3. 行为配置页缺少“鼠标按键绑定层”，导致点击 / 双击 / 长按的触发定义不完整。
4. 光标状态（文本、手型、帮助、问号等）的图标替换能力完全缺失。

这四类缺口中，后 3 类属于功能能力缺口，严重程度高于一般的交互细节问题。

---

# 4. 缺口一：Popup 缺少当前启用方案预览

## 4.1 缺口定义

当前 V3 的 Popup 中已经有：

* 当前状态卡
* 快速切换列表
* 当前网站控制
* 当前网页试用

但缺少一个非常关键的模块：

**当前启用方案的可感知预览。**

用户只能看到：

* 当前方案名称
* 当前方案在快速切换列表中的一行状态

但无法第一眼看到：

* 当前方案的视觉风格
* 当前方案的主要效果预览
* 当前方案是否就是自己想要的那个

## 4.2 为什么这是问题

Popup 是最高频入口。用户每次打开弹窗时，最自然的判断链路是：

1. 现在开没开？
2. 当前网站能不能生效？
3. 我现在用的是哪个方案？
4. 这个方案长什么样？
5. 我要不要换一个？

如果没有当前方案预览，用户只能靠名字做判断，这会降低“方案切换”的直觉性。

## 4.3 影响

* 降低当前方案的可识别性。
* 用户难以快速确认当前风格。
* 方案切换更像列表操作，而不是风格选择。
* Popup 的“有感知价值”下降。

## 4.4 补全方案

建议在 Popup 中增加：

### 模块名称

当前启用方案预览

### 放置位置

主状态卡下方、快速切换区上方

### 模块内容

* 当前方案名称
* 一句话风格摘要
* 小型视觉预览缩略图
* 主效果摘要（如：点击飘字 + 粒子 + 轻音效）

### 模块动作

* 预览当前方案
* 进入编辑

## 4.5 设计要求

* 这是“微预览卡”，不是恢复到 V1 的大方案卡片墙。
* 体积应比 V2 方案大卡更小。
* 要求强感知，但不恢复臃肿。

## 4.6 优先级

P0

因为它直接影响 Popup 第一眼体验质量。

---

# 5. 缺口二：行为配置页缺少真实可操作的素材与音效配置模块

## 5.1 缺口定义

当前 V3 的 Behaviors 页已经有：

* 行为切换
* 行为摘要
* 快速模式
* 高级模式分组标题
* 预览区

但高级模式中的“素材与音效”仍然只是分组标题，尚未展开为真正可操作的配置模块。

用户当前无法真正配置以下高价值内容：

* 飘字文案
* 自定义图片
* 自定义点击音效
* 图片大小与位置
* 音效音量与素材

## 5.2 为什么这是问题

这些不是边缘能力，而是用户最容易理解、最容易产生“我在自定义”的核心能力。

根据 PRD，MVP 首批效果类型明确包含：

* 飘字
* 粒子
* 图片
* 简单动画
* 页面反馈
* 基础音效

并且配置能力明确支持：

* 文案内容（用于飘字）
* 素材选择（用于图片 / 音效）

也就是说，这些不是附加要求，而是产品定义内能力。

## 5.3 当前风险

如果用户进入行为配置页，却无法配置最关心的内容，会形成典型的落差：

* 看起来已经有了“素材与音效”
* 实际却不能配置图片、音效、飘字
* 最终体验变成“看起来能改，其实改不了”

这会严重削弱用户对产品的信任。

## 5.4 补全方案

建议优先把高级模式中的“素材与音效”分组做实。

### 一级模块名称

素材与音效

### 建议结构

分为 4 个子模块：

#### A. 飘字配置

字段：

* 是否启用飘字
* 飘字内容输入框
* 字号
* 上浮距离
* 持续时间
* 随机文案开关（后续）

动作：

* 恢复默认

#### B. 图片效果配置

字段：

* 是否启用图片效果
* 选择内置图片
* 上传自定义图片
* 图片大小
* 偏移方式（跟随鼠标 / 偏移固定值）
* 透明度

动作：

* 更换图片
* 移除图片
* 恢复默认

#### C. 点击音效配置

字段：

* 是否启用音效
* 选择内置音效
* 上传自定义音效
* 音量
* 音效延迟（后续可选）

动作：

* 试听
* 更换音效
* 恢复默认

#### D. 组合效果开关

字段：

* 当前行为是否启用文字
* 是否启用图片
* 是否启用粒子
* 是否启用音效
* 是否启用页面波纹

作用：
让一个行为支持“多种效果同时组合”。

## 5.5 为什么要加“组合效果开关”

因为如果只允许一个行为只能配置单一效果，用户会很难实现“点击一下出现图片 + 飘字 + 音效”这种典型需求。

哪怕 MVP 逻辑上仍以一个主效果为主，UI 也建议预留组合开关概念。

## 5.6 优先级

P0

这是当前 V3 最大的可配置能力缺口之一。

# 6. 缺口三：行为配置页缺少鼠标按键绑定层

## 6.1 缺口定义

当前行为配置页支持的行为维度是：

* 点击
* 双击
* 长按
* 悬停
* 滚轮

但没有任何地方定义：

* 点击对应左键还是右键
* 双击对应左键双击还是右键双击
* 长按对应左键还是右键
* 滚轮是向上、向下还是中键按压

这意味着当前行为定义缺少“触发来源”这一层。

## 6.2 为什么这是问题

对普通用户来说，“点击”默认会理解为左键，但当产品进入深度配置阶段时，如果不支持按键绑定，会出现以下问题：

* 行为定义不完整。
* 高阶用户无法表达“右键触发特效”。
* 一个行为无法进一步区分不同按键来源。
* 页面看起来像支持深配置，但实际上触发逻辑仍然过粗。

## 6.3 与当前文档的关系

PRD 中，P0 的事件触发体系主要覆盖：

* 单击
* 双击
* 长按
* 悬停
* 滚轮

而更多事件（如右键）可放到 P1。

因此从最原始 MVP 边界看，V3 现在的行为列表并不算直接违背旧 P0。

但从今天这版产品定位和用户预期看：

**如果已经进入“行为配置”，就不该继续用过于粗糙的行为定义。**

## 6.4 补全方案

建议在 Behaviors 页的“触发规则”分组中，增加：

### 模块名称

触发按键绑定

### 建议结构

#### A. 点击

* 左键点击
* 右键点击（P1 可开放）
* 中键点击（后续）

#### B. 双击

* 左键双击
* 右键双击（P1 可开放）

#### C. 长按

* 左键长按
* 右键长按（P1 可开放）

#### D. 滚轮

* 向上滚动
* 向下滚动
* 中键按压（后续）

### 控件形式建议

* 单选
* 多选开关（若未来支持一个行为绑定多个触发来源）

## 6.5 MVP 与 P1 划分建议

### MVP（P0）建议补齐

* 点击：左键 / 右键 结构预留
* 双击：默认左键，但 UI 上预留绑定入口
* 长按：默认左键，但 UI 上预留绑定入口
* 滚轮：向上 / 向下

### P1 再开放

* 右键双击
* 中键按压
* 组合按键触发

## 6.6 优先级

P0（至少补 UI 结构）
P1（开放完整高级组合）

因为这是行为定义是否完整的关键一层。

---

# 7. 缺口四：光标状态图标替换能力完全缺失

## 7.1 缺口定义

当前 V3 中完全没有承载“光标状态图标替换”的页面与模块。

缺失内容包括：

* 默认箭头态
* 文本输入态
* 手型链接态
* 帮助态
* 问号态
* 禁止态（后续）

以及这些状态对应的：

* 内置图标选择
* 自定义图片上传
* 图标预览
* 恢复默认

## 7.2 为什么这是严重问题

因为在主 PRD 中，产品定义阶段就已经写明：

产品允许用户将“不同光标状态”等交互行为进行可视化和个性化配置，并支持光标替换。

也就是说：

* 这不是后来新增的小需求。
* 这是产品定义本身就包含的能力方向。

当前 V3 没有任何落点，属于完整能力线缺失。

## 7.3 影响

* 产品定义与页面结构脱节。
* 用户无法配置光标状态替换。
