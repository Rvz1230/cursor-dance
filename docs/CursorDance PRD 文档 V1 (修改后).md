## 2. 产品概述

### 2.1 产品定义

CursorDance 是一款面向 PC 浏览器用户的鼠标特效浏览器插件。产品允许用户将鼠标点击、悬停、滚轮、连击、不同光标状态等交互行为进行可视化和个性化配置，实现音效、飘字、粒子、图片、动画、页面反馈、光标替换等效果，并支持方案保存、预设套用、实时预览和后续分享扩展。

### 2.2 产品愿景

让用户能够以低门槛、高自由度的方式，把原本普通的鼠标交互变成具有趣味性、个性化表达和可玩性的浏览体验。

### 2.3 核心价值

- 提供可感知的鼠标交互反馈，增强浏览乐趣。
- 提供低门槛的预设方案，降低配置难度。
- 提供高自由度编辑器，满足进阶用户创作需求。
- 提供实时预览与调试机制，提升配置效率和成就感。
- 为后续主题包、社区分享、创作者生态和商业化预留空间。

### 2.4 当前阶段产品判断

CursorDance 当前阶段的目标不是覆盖所有鼠标交互场景，而是优先验证一个更核心的判断：

**用户是否愿意在日常浏览中长期保留"鼠标特效反馈"这一体验。**

因此首阶段不是做"最全的鼠标交互特效系统"，而是做"**最好上手、最容易形成记忆点、最容易验证传播性的预设驱动产品**"。

### 3.3 反感场景与负向体验

以下情况是产品设计必须重点规避的：

- 用户在办公、阅读、输入内容时觉得干扰明显
- 用户在录屏、会议、演示时不希望出现杂乱特效
- 高频点击造成卡顿、音效轰炸或页面残留
- 在不适合的站点中持续触发高强度反馈
- 新用户首次进入就面对大量参数，产生畏难情绪

## 4. 产品目标与指标

### 4.1 阶段目标

**MVP 阶段目标：**

- 验证"鼠标点击特效 + 方案编辑 + 预览"的核心体验可行性
- 通过 2\~4 套高辨识度预设验证用户兴趣
- 建立稳定的效果配置模型，为后续迭代打基础

### 4.2 核心指标

- 首次配置成功率
- 预设方案套用率
- 自定义方案保存率
- 日均触发次数
- 7 日留存率
- 方案导出 / 分享率
- 平均预览使用时长
- 静音模式开启率
- 黑名单站点添加率
- 插件总开关j关闭率

### 4.3 MVP 成功判定

MVP 成功不以"功能做得多"为标准，而以以下结果判断：

- 新用户可在 1 分钟内完成首次预设套用
- 新用户可在 5 分钟内完成一次简单自定义并保存
- 至少有明确比例用户进入预览或编辑流程，而不是只套用不修改
- 木鱼修行与游戏连击两套预设显著高于其他预设的使用与传播表现
- 插件整体关闭率、静音率、站点拉黑率维持在可接受范围内

### 4.4 MVP 失败信号

若出现以下现象，应视为产品方向需调整：

- 用户只体验预设，不愿意进入编辑器
- 高频出现关闭插件、静音、加入黑名单站点等行为
- 预览区表现良好，但真实网页中经常不生效或观感差异大
- 用户对"新鲜"有反应，但无法形成持续留存

### 5.3 范围控制原则

本产品配置空间天然很大，因此必须坚持以下原则：

- **MVP 验证"好玩 + 不烦 + 可保存"闭环**，不追求参数全开放

- **首版优先保证运行稳定性和不打扰原则**，不做重度全屏反馈

- 元素级视觉反馈
- 音效反馈
- 不影响页面布局的飘字/粒子效果

### 11.5 预览区与真实网页的差异说明

需要明确：预览区用于模拟效果，不等同于真实网页运行环境。

因此建议在后续设计中补充：

1. **标准舞台预览**：用于稳定、快速调参数
2. **真实页面试运行**：允许用户短时间在当前网页临时试用
3. **问题诊断模式**：显示命中方案、触发事件、当前状态、是否被规则拦截或性能降级

**这样能提前解决一个高概率问题：预览里很好看，到了真实网页里不一定一致。**

### 20.5 打扰风险

- 高强度音效和粒子可能影响办公、阅读与输入体验
- 在不适合的站点中持续触发会增加反感和卸载风险

### 20.6 对应策略

- **预设先行**：通过预设约束效果强度
- **高级模式后置**：复杂配置不默认展示
- **默认启用安全值**：首版参数偏保守
- **提供性能降级与站点黑名单**：用户可控
- **分享配置与资源解耦**：避免外部资源风险
- **默认不拦截原页面逻辑**：不破坏正常使用
- **对输入框、办公场景和弱性能场景采取保守策略**

## 21. 结论

CursorDance 不应被定义为单一"鼠标点击特效插件"，而应被定义为一套"**浏览器内的鼠标交互特效引擎 + 个性化创作系统**"。

但在 MVP 阶段，应更务实地聚焦于：

- 左右键分离配置
- 飘字、音效、图片、粒子、基础动画
- 连击玩法
- 光标替换（default / pointer / text）
- 预设方案
- 实时预览
- 反打扰与性能控制

**在此基础上，优先打造"木鱼修行"和"游戏连击"两套爆点预设，快速验证产品吸引力、传播潜力和持续留存价值。**
