# CursorDance MVP 功能清单与优先级表 V1

## 1. 文档目标

本文件用于在《CursorDance PRD 文档 V1》的基础上，进一步明确 MVP 阶段的功能边界、优先级、交付目标、依赖关系与版本节奏，帮助产品、设计、前端和插件工程实现统一理解，并作为后续排期评审与开发拆解依据。

---

## 2. MVP 定义

CursorDance MVP 的目标不是一次性做全所有鼠标交互能力，而是优先验证以下核心命题：

1. 用户是否会对"鼠标点击特效"产生持续兴趣
2. 用户是否愿意套用或自定义方案
3. 左右键差异化配置是否具备足够感知价值
4. 预设 + 预览 + 简单编辑的组合是否足够顺畅
5. 产品能否在浏览器插件环境下稳定运行，并保持较好的性能与兼容性

因此，MVP 阶段聚焦于：

- 可感知的点击反馈
- 低门槛的配置能力
- 强传播性的预设方案
- 足够好用的预览与编辑体验
- 可扩展的数据模型与运行时引擎基础

**同时，MVP 不以"支持多少参数"为目标，而以"能否形成可复用、可保存、可传播、可持续使用的最小闭环"为目标。**

---

## 5. MVP 功能详细清单

### 5.1 插件基础能力

#### 5.1.1 插件总开关

- **优先级**：P0
- **说明**：允许用户一键启用或禁用插件
- **必要性**：核心基础能力
- **依赖**：无
- **不做什么**：不做定时开关、深夜模式等高级开关
- **验收标准**：开关状态可实时生效，不影响页面原有逻辑

#### 5.1.2 渲染容器管理

- **优先级**：P0
- **说明**：管理特效渲染的 DOM 容器生命周期
- **必要性**：所有视觉特效的渲染载体
- **依赖**：页面注入机制
- **不做什么**：不做多容器管理、容器层级自定义
- **验收标准**：容器可正常创建、销毁、清理，无页面污染

#### 5.1.3 方案存储与读取

- **优先级**：P0
- **说明**：本地存储方案数据，支持保存、读取、删除
- **必要性**：用户配置持久化的基础
- **依赖**：Chrome Storage API
- **不做什么**：不做云端同步、多设备同步
- **验收标准**：方案可正常保存 30 天以上，数据不丢失

### 5.2 鼠标事件监听

#### 5.2.1 点击事件监听与分流

- **优先级**：P0
- **说明**：监听左键和右键点击行为，并分发给对应方案执行
- **必要性**：核心交互基础
- **依赖**：运行时引擎、当前方案解析
- **不做什么**：首版不覆盖中键、长按、拖拽等全套事件体系
- **验收标准**：左键与右键可触发各自独立效果

#### 5.2.2 连续点击计数

- **优先级**：P1
- **说明**：在限定时间窗口内统计连续点击次数
- **必要性**：连击玩法的基础
- **依赖**：点击事件监听、时间窗口管理
- **不做什么**：不做跨时间窗口累计、全球排名
- **验收标准**：连续点击时计数正确，超时后可重置

#### 5.2.3 Cursor 状态监听

- **优先级**：P2
- **说明**：监听鼠标在不同元素上的 cursor 状态变化
- **必要性**：光标替换的前置能力
- **依赖**：鼠标事件监听
- **不做什么**：首版只做 default / pointer / text 三种状态
- **验收标准**：状态切换可被正确识别并触发对应效果

### 5.3 效果配置

#### 5.3.1 文案飘字效果

- **优先级**：P0
- **说明**：点击后出现飘动文字
- **MVP 范围**：
  - 文案内容
  - 文案颜色
  - 字号
  - 字重
  - 透明度
  - 显示位置
  - 飘动方向
  - 动画时长
- **依赖**：渲染容器、动画引擎
- **不做什么**：不做随机文案池、渐变文字、描边阴影等高级效果
- **验收标准**：文案可正常显示、飘动、消失，无视觉残留

#### 5.3.2 图片 / 图标效果

- **优先级**：P1
- **说明**：点击后显示图片或图标
- **MVP 范围**：
  - 图片资源引用
  - 图片尺寸
  - 图片位置偏移
  - 动画时长
  - 透明度
- **依赖**：资源加载、图片渲染
- **不做什么**：不做序列帧、多图轮播、GIF 支持
- **验收标准**：图片可正常加载显示，位置正确，无闪烁

#### 5.3.3 音效配置

- **优先级**：P0
- **说明**：支持点击音效
- **MVP 范围**：
  - 音频资源
  - 音量
  - 播放速度
  - 是否允许重叠播放
- **依赖**：音频播放管理、静音模式
- **不做什么**：不做多轨混音和复杂分层音效系统
- **验收标准**：可在预览与实际页面中正常播放，无严重延迟

#### 5.3.4 基础动画效果

- **优先级**：P1
- **说明**：提供基础的动画能力
- **MVP 范围**：
  - 动画类型（淡入淡出、缩放、飘动）
  - 动画时长
  - 动画曲线
  - 动画方向
- **依赖**：动画引擎
- **不做什么**：不做自定义关键帧、骨骼动画
- **验收标准**：动画流畅，无卡顿，可被中断

#### 5.3.5 粒子效果

- **优先级**：P1
- **说明**：点击后产生粒子扩散效果
- **MVP 范围**：
  - 粒子形状
  - 粒子颜色
  - 粒子数量
  - 粒子扩散范围
  - 粒子生命周期
- **依赖**：粒子系统渲染
- **不做什么**：不做复杂粒子物理、多层叠加、粒子间交互
- **验收标准**：粒子效果可正常显示，数量和范围可接受，不影响性能

#### 5.3.6 基础连击配置

- **优先级**：P1
- **说明**：支持限定时间窗口内的连续点击反馈
- **MVP 范围**：
  - 连击时间窗口
  - 连击计数显示
  - 连击阈值映射（至少 3 个档位）
  - 不同档位差异化反馈
- **依赖**：事件计数、状态重置、阈值映射
- **不做什么**：不做复杂暴击、终结技、跨场景连击体系
- **验收标准**：连续点击时计数正确，超时后可重置；至少 3 个阈值档位具备明显差异反馈

#### 5.3.7 光标替换

- **优先级**：P2
- **说明**：支持不同状态下的光标替换
- **MVP 范围**：
  - default 状态
  - pointer 状态
  - text 状态
- **依赖**：Cursor 状态监听
- **不做什么**：首版不做自定义光标图片、动态光标
- **验收标准**：三种状态可正确切换，无闪烁

### 5.4 预设方案

#### 5.4.1 官方预设

- **优先级**：P0
- **说明**：提供预置的方案供用户直接使用
- **MVP 范围**：
  - 木鱼修行预设
  - 游戏连击预设
  - 至少 1~2 套基础预设
- **依赖**：方案存储、效果渲染
- **不做什么**：不做预设商店、用户上传预设
- **验收标准**：预设可一键套用，效果完整

#### 5.4.2 预设展示与切换

- **优先级**：P0
- **说明**：展示预设列表，支持快速切换
- **依赖**：预设存储、UI 组件
- **不做什么**：不做预设评分、评论、收藏
- **验收标准**：预设列表清晰，切换流畅

#### 5.4.3 预设应用

- **优先级**：P0
- **说明**：将预设应用到当前网页
- **依赖**：方案解析、效果执行
- **不做什么**：不做预设对比、预设克隆
- **验收标准**：应用后效果立即生效

### 5.5 编辑器

#### 5.5.1 基础编辑器界面

- **优先级**：P0
- **说明**：提供简单易用的配置界面
- **MVP 范围**：
  - 文案配置
  - 音效配置
  - 基础动画配置
- **依赖**：UI 框架、状态管理
- **不做什么**：不做全功能编辑器、代码编辑器
- **验收标准**：界面简洁，参数一目了然，新用户可快速理解

#### 5.5.2 参数即时生效

- **优先级**：P1
- **说明**：调整参数后效果立即更新
- **依赖**：实时预览、状态同步
- **不做什么**：不做版本历史、参数撤销
- **验收标准**：调整与预览延迟小于 300ms

#### 5.5.3 方案保存

- **优先级**：P0
- **说明**：保存用户自定义的配置方案
- **依赖**：方案存储
- **不做什么**：不做自动保存、云端同步
- **验收标准**：保存后可正常读取恢复

#### 5.5.4 方案管理

- **优先级**：P2
- **说明**：支持方案的删除、复制、编辑基础信息
- **MVP 范围**：
  - 方案删除
  - 方案复制
  - 方案重命名
- **依赖**：方案存储
- **不做什么**：不做方案导入导出（JSON 除外）
- **验收标准**：操作后状态正确，无残留

#### 5.5.5 JSON 导入导出

- **优先级**：P2
- **说明**：支持方案的 JSON 格式导出和导入
- **依赖**：方案序列化
- **不做什么**：不做在线模板市场
- **验收标准**：导出的 JSON 可被正常导入并使用

### 5.6 预览区

#### 5.6.1 标准舞台预览

- **优先级**：P0
- **说明**：在稳定环境中模拟效果
- **依赖**：渲染容器、模拟点击
- **不做什么**：不做多舞台切换
- **验收标准**：可快速调参，效果稳定

#### 5.6.2 真实页面试运行

- **优先级**：P1
- **说明**：允许在当前网页临时试用
- **依赖**：运行时注入
- **不做什么**：不做全域持久化
- **验收标准**：试用效果与预览基本一致

#### 5.6.3 问题诊断模式

- **优先级**：P2
- **说明**：显示命中方案、触发事件、当前状态等
- **MVP 范围**：
  - 命中方案显示
  - 触发事件记录
  - 规则拦截提示
- **依赖**：运行时状态监控
- **不做什么**：不做复杂日志、远程上报
- **验收标准**：问题可被快速定位

### 5.7 控制与保护

#### 5.7.1 静音模式

- **优先级**：P0
- **说明**：一键关闭所有音效
- **依赖**：音频播放管理
- **不做什么**：不做定时静音、场景化静音
- **验收标准**：可快速开启关闭，状态持久

#### 5.7.2 性能模式

- **优先级**：P1
- **说明**：降低动画复杂度，保证性能
- **MVP 范围**：
  - 降低粒子数量
  - 简化动画
  - 延长连击窗口
- **依赖**：效果引擎降级
- **不做什么**：不做极致性能优化、自定义降级规则
- **验收标准**：开启后帧率明显提升

#### 5.7.3 降低动画模式

- **优先级**：P1
- **说明**：减少或关闭动画效果
- **依赖**：动画控制
- **不做什么**：不做动画速度调节
- **验收标准**：动画可正常关闭，无残留

#### 5.7.4 黑名单站点

- **优先级**：P1
- **说明**：指定站点禁用特效
- **MVP 范围**：
  - 站点添加
  - 站点移除
  - 站点列表展示
- **依赖**：URL 匹配
- **不做什么**：不做站点分组、智能推荐
- **验收标准**：黑名单站点中特效不触发

## 6. 推荐开发顺序

### 阶段 1：跑通基础闭环

**目标**：实现"选预设 → 点击生效 → 可保存"的最小闭环

**建议包含**：

- 插件总开关
- 渲染容器
- 左键 / 右键监听
- 文案飘字
- 音效
- 官方预设展示
- 预设应用
- 保存方案

**阶段演示标准**：至少可用两套简单预设完成一键应用，并在真实网页中稳定生效。

### 阶段 2：补齐可玩性

**目标**：形成明显差异化体验

**建议包含**：

- 图片 / 图标效果
- 粒子效果
- 基础动画
- 木鱼预设
- 游戏连击预设
- 预览区
- 参数即时生效

**阶段演示标准**：木鱼与游戏连击两套预设可完整演示，且预览区与真实效果基本一致。

### 阶段 3：补齐可配置性

**目标**：从"能用"提升为"好用"

**建议包含**：

- 连击配置
- 光标替换（常用状态）
- JSON 导入导出
- 黑名单站点
- 降低动画模式
- 方案删除 / 复制 / 编辑基础信息

**阶段演示标准**：用户可完成一次轻量编辑、预览、保存、导出再导入的完整闭环。

### 阶段 4：MVP 收口上线

**目标**：完成保护能力、兼容性与体验收尾

**建议包含**：

- 静音 / 性能模式完善
- 黑名单稳定性验证
- 兼容性与性能修复
- 首批预设质感打磨

**阶段演示标准**：在常见网页环境中运行稳定，具备上线可用性。

## 8. MVP 验收标准

### 8.1 功能层面

- 预设可一键套用并生效
- 文案、音效、图片、粒子效果可正常触发
- 连击计数正确，阈值反馈有明显差异
- 方案可保存、读取、删除
- JSON 可导出导入
- 黑名单站点可正常生效

### 8.2 体验层面

- 参数修改后预览及时反馈
- 界面结构清晰，不让新用户迷失
- 基础编辑流程可理解
- 插件对正常浏览干扰较低
- 静音、性能模式、黑名单入口清晰可用

### 8.3 技术层面

- 常规网页中连续点击 30 次以上不出现严重卡顿
- 音频播放无明显错乱
- 特效实例可正常回收
- 默认场景下 CPU / 内存开销可控
- 方案切换后旧状态不会持续残留
- 资源缺失或格式异常时有明确兜底逻辑

---

## 9. 最终建议

CursorDance 的 MVP 不应做成"复杂参数编辑器"，而应做成"**预设驱动的鼠标特效体验产品**"。

因此，首版最关键的是：

1. **先让用户一键套用就觉得好玩**
2. **再让用户轻松改出属于自己的版本**
3. **用预览区降低配置门槛**
4. **用木鱼和 combo 两个爆点场景验证传播性**
5. **用静音、性能模式、黑名单等能力控制打扰感**

如果 MVP 能把这五件事做好，后续再扩展光标状态、站点规则、主题包和社区分享，成功率会更高。
